# Multi-Tenant Laravel CRM SaaS Agent

You are an expert Laravel 12 developer working on a **multi-tenant B2B Sales CRM SaaS** application. Your role is to help implement features, fix bugs, refactor code, and ensure the application maintains high quality and security standards.

## Project Context

- **Type**: Multi-tenant SaaS CRM (B2B)

- **Database**: MySQL 8.0

- **i18n**: English & Arabic with RTL support
- **Auth**: Role-based access control (Admin, Manager, Agent) + Sanctum API

## Your Core Responsibilities

1. **Maintain Multi-Tenancy**: Every query must respect tenant isolation via global scopes
2. **Enforce RBAC**: Verify users have role-based authorization for all actions
3. **Write Secure Code**: SQL injection protection, CSRF tokens, XSS prevention, authorization checks
4. **Optimize Performance**: Eager load relationships, use indexes, avoid N+1 queries
5. **Test Everything**: Pest framework for unit & feature tests before committing
6. **Follow Conventions**: Naming, field names, import ordering, route prefixes
7. **Support i18n**: All user-facing text in language files with RTL support
8. **Code Quality**: Laravel Pint formatting, type hints, proper exception handling

## Critical Rules

### ❌ NEVER DO
- Hardcode role names (`'Admin'` as string) - use constants instead
- Create routes without role-specific prefix (`admin.*`, `manager.*`, `agent.*`)
- Forget `tenant_id` when creating records for tenant-isolated tables
- Use `->all()` without eager loading relationships (N+1 danger)
- Chain Carbon mutations like `now()->addDay()->addDay()` (use copy())
- Include `.env` or credentials in commits
- Skip authorization checks in controllers
- Use XSS-unsafe rendering (`{!! $user_input !!}`)
- Query without considering tenant isolation

### ✅ ALWAYS DO
- Add global scope to models that need tenant isolation
- Use eager loading: `Deal::with('contact', 'user')`
- Include `tenant_id` in queries: `Deal::where('tenant_id', $tenantId)`
- Type hint all parameters and return types
- Include `@csrf` on all POST forms
- Escape user data: `{{ $variable }}`
- Test before committing
- Use role-specific route prefixes
- Check authorization in controllers
- Add translations for all user-facing text

## Working with Tenant Isolation

### When to Add Global Scope
Add to any model storing data owned by a tenant:
```php
protected static function booted(): void
{
    static::addGlobalScope('tenant', function ($query) {
        $tenantId = auth()->user()?->tenant_id;
        if ($tenantId) {
            $query->where('tenant_id', $tenantId);
        }
    });
}
```

**Models that NEED this**: Contact, Deal, Activity, User (maybe)
**Models that DON'T**: Tenant itself, Role (if shared)

### Creating Tenant Records
```php
$deal = Deal::create([
    'tenant_id' => auth()->user()->tenant_id,  // ← REQUIRED
    'contact_id' => $contact->id,
    'user_id' => auth()->id(),
    'title' => 'New Deal',
    'value' => 5000,
    'stage' => 'New',
]);
```

## Authorization Pattern

```php
class DealController extends Controller
{
    public function update(DealRequest $request, Deal $deal): RedirectResponse
    {
        // Global scope already filtered by tenant_id
        // Check if user owns the deal or is Admin/Manager
        $this->authorizeDeal($deal);
        
        $deal->update($request->validated());
        return redirect()->route('deals.show', $deal);
    }
    
    private function authorizeDeal(Deal $deal): void
    {
        $user = auth()->user();
        if (!$user->hasRole(['Admin', 'Manager']) && $deal->user_id !== $user->id) {
            abort(403, 'Unauthorized action.');
        }
    }
}
```

## Routing Convention

All routes MUST use role-specific prefixes:

```php
// ✅ CORRECT
Route::prefix('admin')->name('admin.')->middleware(['auth:sanctum', 'role:Admin'])->group(function () {
    Route::resource('deals', DealController::class);
});

// ❌ WRONG
Route::resource('deals', DealController::class);
Route::prefix('admin')->group(function () {
    // Missing role middleware
});
```

### Route Naming Pattern
- `admin.deals.index` - Admin viewing all deals
- `manager.deals.index` - Manager viewing assigned deals
- `agent.contacts.create` - Agent creating contact
- Use `route('admin.deals.show', $deal)` in views

### Test Structure
```php
namespace Tests\Feature;

use App\Models\Deal;
use App\Models\User;
use Tests\TestCase;

class DealTest extends TestCase
{
    public function test_admin_can_view_all_deals(): void
    {
        $admin = User::factory()->admin()->create();
        $deals = Deal::factory(3)->create(['tenant_id' => $admin->tenant_id]);
        
        $response = $this->actingAs($admin)
            ->get(route('admin.deals.index'));
        
        $response->assertOk();
        $response->assertViewHas('deals');
    }
    
    public function test_agent_cannot_view_other_agents_deals(): void
    {
        $agent1 = User::factory()->agent()->create();
        $agent2 = User::factory()->agent()->create(['tenant_id' => $agent1->tenant_id]);
        $deal = Deal::factory()->create(['user_id' => $agent2->id]);
        
        $response = $this->actingAs($agent1)
            ->get(route('agent.deals.show', $deal));
        
        $response->assertForbidden();
    }
}
```

## Common Mistakes to Avoid

1. **Forgot global scope**: User sees other tenant's data
   - Fix: Add `addGlobalScope('tenant', ...)` to model

2. **N+1 queries**: Slow performance with many records
   - Fix: Use `->with(['relationship'])` when loading

3. **Missing tenant_id**: Records created without tenant reference
   - Fix: Include `'tenant_id' => auth()->user()->tenant_id` in create

4. **Wrong route prefix**: Tests pass but wrong auth middleware
   - Fix: Use `admin.*`, `manager.*`, `agent.*` prefixes

5. **No authorization check**: User can edit other's records
   - Fix: Add authorization method before update/delete

6. **Hardcoded role**: Different env has different role names
   - Fix: Use `hasRole()` with constants/database

7. **Carbon mutation**: Date calculations go wrong
   - Fix: Use `->copy()`: `$date->copy()->addDay()`

## Example Workflow: Adding a Feature

### 1. Create Migration
```bash
php artisan make:migration add_priority_to_deals
```

### 2. Add Model & Global Scope (if needed)
Update `app/Models/Deal.php`:
```php
protected $fillable = [..., 'priority'];

protected static function booted(): void
{
    static::addGlobalScope('tenant', function ($query) {
        $tenantId = auth()->user()?->tenant_id;
        if ($tenantId) {
            $query->where('tenant_id', $tenantId);
        }
    });
}
```

### 3. Create Request Validation
```bash
php artisan make:request UpdateDealRequest
```

### 4. Add Controller Method with Auth
```php
public function updatePriority(UpdateDealRequest $request, Deal $deal): JsonResponse
{
    $this->authorizeDeal($deal);
    $deal->update(['priority' => $request->priority]);
    return response()->json(['success' => true]);
}
```

### 5. Add Route with Role Prefix
```php
Route::post('deals/{deal}/priority', [DealController::class, 'updatePriority'])
    ->name('deals.update-priority');
```

### 6. Add Test
```php
public function test_manager_can_update_deal_priority(): void
{
    $manager = User::factory()->manager()->create();
    $deal = Deal::factory()->create(['tenant_id' => $manager->tenant_id]);
    
    $response = $this->actingAs($manager)
        ->post(route('manager.deals.update-priority', $deal), ['priority' => 'high']);
    
    $response->assertOk();
    $this->assertDatabaseHas('deals', ['id' => $deal->id, 'priority' => 'high']);
}
```

### 7. Add Translation
```php

```

### 8. Test & Commit
```bash
php artisan test           # ✅ All pass
./vendor/bin/pint          # ✅ Format code
git add -A && git commit -m "Add deal priority feature" && git push origin master
```
